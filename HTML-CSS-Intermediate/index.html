WHAT IS FLEXBOX?

FLEXBOX: WHAT IT IS?

  • In the past we have use floats to build column layouts, but Flexbox is now considered one of the best alternatives for grid layouts.
  • Flexbox help us create incredible structures in our page and solves a lot of layout issues that float either causes or can’t solve (or both).

FLEXBOX: WHAT IT IS?

Before we start with describing the flexbox properties let’s give a little introduction of the flexbox model. The flex layout is constituted of parent container referred as flex container and its immediate children which are called flex items.

FLEXBOX: WHAT IT IS?

Parent wrapper: flex container

FLEXBOX: WHAT IT IS?

Direct children: flex items

HOW DOES FLEXBOX WORK?

STEPS TO ACHIEVE A MULTI-COLUMN LAYOUT

FLEX CONTAINER: Add display:flex

We can set display: flex on the parent container to start the flex magic. By default, all direct children will become flex items and will be placed side-by-side on the main axis.

FLEX ITEMS: Set a width in percentages

flex-basis can be used to determine the width for flex items in percentages.

FLEXBOX: HOW DOES FLEXBOX WORK?

  • By default, items are arranged along the main axis, from left to right. This is why your squares defaulted to a horizontal line once you applied display:flex;

FLEX DIRECTION

  • flex-direction, let’s you rotate the main axis.
  • There’s an important distinction to make here: flex-direction: column doesn’t align the squares on the cross axis instead of the main axis. It makes the main axis itself go from horizontal to vertical.

JUSTIFY CONTENT

  • justify-content, controls how you align items on the main axis.
  • There are five commands available to use with justify-content:
    • flex-start
    • flex-end
    • center
    • space-around
    • space-between

ALIGN CONTENT

  • align-items, controls how you align items on the cross axis.
  • There are five commands available to use with align-items:
    • flex-start
    • flex-end
    • center
    • stretch
    • baseline

ROWS WITH FLEXBOX

Since all our rows will be set to display:flex; we can store this property in a class and apply it to any element to turn it into a row.

COLUMNS WITH FLEXBOX

We can create classes for the different options for column widths. We can then apply these classes to any flex item to set its width.

MEDIA QUERIES

MEDIA QUERIES: VIEWPORT META TAG

  • The viewport meta tag controls how a web page is displayed on a mobile device.
  • Without the tag, mobile devices will assume you want the full desktop experience and will set the viewport width at 980px (iOS)

DEVICE-WIDTH

  • This tells the browser “My Website adapts to your width”

INITIAL-SCALE

  • Sets the initial zoom level and prevents default zooming

MEDIA QUERIES

Media queries allow us to add or change styles for elements based on:

  • screen size (width or height of browser window)
  • device orientation (portrait or landscape)
  • display density (retina or non-retina)

With media queries, we can allow most of our styles to remain the same, while we make small tweaks for specific formats.

ADDING MEDIA QUERIES

MEDIA QUERIES: SYNTAX

MEDIA QUERIES: SYNTAX

MEDIA QUERIES: ORDER

  • Place your media queries at the bottom of your stylesheet so they can override code that comes before them.
  • If you have multiple media queries, they should go from largest to smallest (max-width: 600 would be placed above max-width: 400)

CSS POSITIONING

CSS POSITIONING

  • By default, elements on a page are similar to these wooden blocks.
  • They will stack one on top of the other in the same order that they are placed in an HTML file. This is referred to as the "normal flow" of a document.

CSS POSITIONING

  • We can use the position property in our CSS to take elements out of the normal flow of the document and specify where they should appear.

CSS POSITIONING - STATIC

  • The default position on each element is static.
  • Elements with a position of static will appear in order and stack on top of each other, like we would expect.

CSS POSITIONING: FIXED

  • Positioned in relation to the browser window
  • When the user scrolls, it stays in the same place.
  • Use right, top, left and bottom properties to specify where the element should go in relation to the browser window.

CSS POSITIONING: RELATIVE

  • Moves an element relative to where it would have been in normal flow.
  • For example: left: 20px adds 20px to an element's left position

CSS POSITIONING: ABSOLUTE

  • Element is taken out of the normal flow of the document.
  • No longer affects the position of other elements on the page (they act like it's not there).
  • You can add the right, top, left and bottom properties to specify where the element should appear

Z-INDEX

OVERLAPPING ELEMENTS: Z-INDEX

  • With relative, absolute, and fixed positioning, elements can overlap.
  • We can use z-index to control which elements are layered on top of each other.
  • This property takes a number — the higher the number the closer that element is to the front.

RESPONSIVE FRAMEWORKS

RESPONSIVE FRAMEWORKS

  • There are many existing CSS Frameworks that offer responsive design.
  • Each framework has its own strengths and weaknesses, and specific areas of application, allowing you to choose based on the needs of a specific project.
  • The aim of frameworks is to provide a common structure so taht developers don’t have to redo it from scratch and can reuse the code provided.
  • Below are some examples of what do the frameworks provide:
    • Fluid grids
    • Flexible images
    • Responsive media queries
    • Styles for buttons and forms, because they're a pain!
    • Lots of base styles like .pull-right, .btn, etc.

CSS PREPROCESSORS

CSS PREPROCESSORS: WHAT IS IT?

  • CSS preprocessors take code written in the preprocessed language and then convert that code into the same old css we’ve been writing for years.
  • Since we're not writing straight CSS, we're not limited to the restrictions of the language.

CSS PREPROCESSORS: WHY IS IT USEFUL?

  • Avoid duplicate code
  • Nested code
  • Increase of productivity
  • Helps to have system-wide definitions
  • Ability to write functions and have variables
  • Allow DRY code (DRY=Don’t Repeat Your Code) which is a good programming practice
  • Error reporting
  • No worries with cross-browser compatibility

CSS PREPROCESSORS: CODE EXAMPLES

CSS PREPROCESSORS: CODE EXAMPLES

CSS PREPROCESSORS: WHAT DO THEY DO?

  • Mixins – Classes for classes.
  • Parametric mixins – Classes to which you can pass parameters, like functions.
  • Nested Rules – Classes within classes, which cut down on repetitive code
  • Operations – Math within CSS
  • Color functions – Edit your colors
  • Namespaces – Groups of styles that can be called by references
  • Scope – Make local changes to styles
  • JavaScript evaluation – JavaScript expressions evaluated in CSS
