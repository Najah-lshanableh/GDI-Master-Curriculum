USING INTERNAL CSS

BOX MODEL

CSS: BOX MODEL

  • All HTML elements can be considered as boxes. In CSS, the term "box model" is used when talking about design and layout.
  • The CSS box model is essentially a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content.
  • Parts:
    • Content - The content of the box, where text and images appear
    • Padding - Clears an area around the content. The padding is transparent
    • Border - A border that goes around the padding and content
    • Margin - Clears an area outside the border. The margin is transparent

CSS CASCADING NATURE

CSS Cascading Nature

  • CSS is an acronym for Cascading Style Sheets, which indicates the importance of cascade.
  • Cascading indicates that the order of CSS rules matter. Which selectors win out in the cascade depends on the following factors such as specificity, source order or importance.

CLASSES & IDS: CSS Cascading Nature

How classes and IDs fit on the CSS cascading nature?

  • CSS rules cascade downward until they are canceled out by another rule. In other words, it goes from general to more specific.
  • CSS rules are able to override one another and cancel each other out, depending on their order.
  • The more specific rule will take precedence over the more general rule.

CSS Cascading: Specificity

  • Specificity is the means by which browsers decide which css property values are the most relevant to an element and, therefore, will be applied. If there are two or more conflicting CSS rules that point to the same element, the browser follows some rules to determine which one is most specific and therefore wins out.
  • It’s usually the reason why your CSS-rules don’t apply to some elements, although you think they should.
  • In other words, think of specificity as a score or rank that determines which style declaration are ultimately applied to an element (i.e. the ID selectors are highly specific comparing to a universal selector).

DISPLAY

DISPLAY: BLOCK LEVEL ELEMENTS

Block-level elements will always start on a new line.

  • Example:
    • <h1>-<h6></h6>
    • <ul>
    • <li>
    • <p>
    • <ol>
    • <div>

DISPLAY: INLINE ELEMENTS

Inline elements will always appear to continue on the same line as their neighboring elements.

  • Example:
    • <a>
    • <image>
    • <span>
    • <em>

KEY DIFFERENCES BETWEEN BLOCK AND INLINE ELEMENTS

  • The most noticeable properties are width, height, margin and padding.
  • If you try to add dimensions to an inline element, some properties will be applied but others no or will partially be applied.

HOW TO CHANGE ELEMENTS DISPLAY INLINE OR BLOCK?

  • You can change whether an element is display as inline or block element by using the display property.

DISPLAY: INLINE AND INLINE-BLOCK

  • When we change the display of an element to be inline, the elements will sit next to each other. But if you use the property display: inline; you can’t set a width, height or margin and padding on top and bottom.
  • In order to be able to set a width, height and margin, we can use a property call display: inline-block;

DISPLAY: NONE

  • If your goal is to hide an element, the property display: none; will hide the elements from the page.

INTRO FLEXBOX

FLEXBOX: WHAT IT IS?

  • In the past we have use floats to build column layouts, but Flexbox is now considered one of the best alternatives for grid layouts.
  • Flexbox help us create incredible structures in our page and solves a lot of layout issues that float either causes or can’t solve (or both).

FLEXBOX: INTRO

  • The flex layout is constituted of parent container referred as a flex container and its immediate children which are called flex items.

FLEXBOX: INTRO

FLEXBOX: HOW DOES FLEXBOX WORK?

  • By default, items are arranged along the main axis, from left to right. This is why your squares defaulted to a horizontal line once you applied display:flex;

FLEX CONTAINER: PARENT WRAPPER

  • In order to get started with Flexbox, you need to make your container into a flex container like the example below:

FLEX DIRECTION

  • flex-direction, let’s you rotate the main axis.
  • There’s an important distinction to make here: flex-direction: column doesn’t align the squares on the cross axis instead of the main axis. It makes the main axis itself go from horizontal to vertical.

JUSTIFY CONTENT

  • justify-content, controls how you align items on the main axis.
  • There are five commands available to use with justify-content:
    • flex-start
    • flex-end
    • center
    • space-around
